# Behavioral Contracts ‚Äî OpenMonoAgent.ai

## Surfaces Covered

- **Host shell CLI** ‚Äî `openmono` bash script on the host; 11 subcommands for Docker lifecycle, config management, and agent launch.
- **Agent REPL / TUI** ‚Äî in-container `openmono` binary; 14 slash commands plus an interactive REPL in TUI mode (full-screen ANSI) or classic mode (scrolling terminal).
- **Tool execution pipeline** ‚Äî 12-step per-tool-call pipeline governing permission, caching, hooks, and artifact persistence; the core trust model.
- **Permission and authorization model** ‚Äî capability-based + legacy pattern-matching permission system with interactive prompting.
- **Sub-agent feature** ‚Äî inline agentic loop (not a full ConversationLoop) with filtered tool set and fresh session state.
- **Configuration surface** ‚Äî `settings.json` (user + project levels), environment variables, and CLI flags; merged in defined priority order.
- **Playbook automation** ‚Äî YAML-defined multi-step workflows loadable by name from REPL or by LLM tool call.

### Surface: Tool Execution Pipeline (arch-CF1 closure)

The 12-step pipeline executes in strict sequential order. Every tool call, regardless of source (LLM, sub-agent, slash command), traverses this pipeline before any side-effecting action occurs.

#### Pipeline step map

| Step | Action | Return on failure | Notes |
|---|---|---|---|
| 1 | Parse `call.Arguments` as JSON | `ToolResult.Error(invalid_json_message)` | Journal: `RecordSchemaRejected` |
| 2 | Schema validation (`SchemaValidator.Validate`) | `ToolResult.Error(schema_error)` | Journal: `RecordSchemaRejected` |
| 3 | Sanity check (`SanityCheck.Check`) | `ToolResult.Error(sanity_error)` | Path-escape detection; workspace boundary enforcement |
| 4 | Plan mode guard | `ToolResult.Error("Plan mode is active‚Ä¶")` | Only for non-read-only tools when `SessionState.Meta.PlanMode = true` |
| 5a | Capability check (`PermissionEngine.CheckCapabilitiesAsync`) | `ToolResult.Error("Permission denied‚Ä¶")` | Primary path; used when `tool.RequiredCapabilities(input).Count > 0` |
| 5b | Legacy permission (`PermissionEngine.CheckAsync`) | `ToolResult.Error("Permission denied‚Ä¶")` | Fallback when tool declares no capabilities |
| 6 | Result cache lookup (`ToolResultCache.TryGet`) | n/a ‚Äî miss is not a failure; proceeds to step 7 | Read-only tools only; validates file mtime + size on hit |
| 7 | Pre-tool hook (`HookRunner.RunPreToolUseHooksAsync`) | Hook failure logged as warning; tool execution **not** blocked | 30 s timeout per hook; failure is non-fatal |
| 8 | Tool execute (`tool.ExecuteAsync`) | `OperationCanceledException` ‚Üí `ToolResult.Cancelled`; any other exception ‚Üí `ToolResult.Crash(msg, retryHint)` | Journal: `RecordToolStarted` before call |
| 9 | Post-tool hook (`HookRunner.RunPostToolUseHooksAsync`) | Hook failure logged as warning; result **not** modified | 30 s timeout; failure is non-fatal; receives `{{tool_output}}` |
| 10 | Artifact store (`ArtifactStore.PersistAndReplace`) | Result returned unmodified if threshold not exceeded | Only if `result.Class == Success` AND `ModelPreview.Length > 50_000` chars |
| 11 | Cache write (`ToolResultCache.Put`) | n/a | Read-only tools on `ResultClass.Success` only |
| 12 | Cache invalidation | n/a | For `FileWrite`, `FileEdit`, `ApplyPatch` only: invalidates `ToolResultCache` + `FileReadTool` internal cache by resolved absolute path |

*observed fact ‚Äî `ConversationLoop.ExecuteSingleToolAsync:627-781`, `HookRunner.cs`, `ArtifactStore.cs`, `ToolResultCache.cs`.*

#### Cache key construction

`{toolName}:{first_16_hex_chars_of_SHA256(NormalizeJson(input))}`

`NormalizeJson` recursively serializes the input JSON with object keys sorted by ordinal string comparison (case-sensitive), no indentation. *observed fact ‚Äî `ToolResultCache.BuildCacheKey`, `WriteNormalized`.*

#### Artifact store threshold

**50,000 chars** (`ArtifactStore.DefaultLargeOutputThreshold = 50_000`). The architecture map cited 20,000 ‚Äî that was an error; 50,000 is the authoritative value from source code. When exceeded, `ModelPreview` is truncated to first 20 lines + last 10 lines with omission note `[N lines omitted ‚Äî full output in artifact {id} ({bytes} bytes)]`. Full content written to `~/.openmono/artifacts/{sessionId}/{toolName}_{yyyyMMddHHmmss}_{hash[..8]}.txt`. *observed fact ‚Äî `ArtifactStore.cs:10`, `BuildTruncatedPreview`.*

#### Tool result return shapes

| `ResultClass` | Factory method | Meaning | `IsError` | Has `RetryHint` |
|---|---|---|---|---|
| `Success` | `ToolResult.Success(content)` | Normal completion | false | no |
| `InvalidInput` | `ToolResult.Error(msg)` | Schema/sanity/permission error | true | no |
| `PermissionDenied` | `ToolResult.PermissionDenied(msg)` | Explicit permission denial | true | optional |
| `StateConflict` | `ToolResult.StateConflict(msg, hint)` | Optimistic concurrency conflict | true | yes |
| `Crash` | `ToolResult.Crash(msg, hint)` | Unhandled exception in tool body | true | yes |
| `Empty` | `ToolResult.Empty(msg)` | Tool returned no output | true | no |
| `Cancelled` | `ToolResult.Cancelled()` | `OperationCanceledException` caught | true | no |

`ToolResult.IsError` is `true` for all non-`Success` classes. The LLM receives `result.Content` (`= result.ModelPreview`) as the tool result string regardless of class. *observed fact ‚Äî `ToolResult.cs`.*

**Owner:** `Session/ConversationLoop.ExecuteSingleToolAsync`, `Session/ArtifactStore`, `Session/ToolResultCache`, `Hooks/HookRunner`, `Tools/ToolResult.cs`.

### Surface: Permission and Authorization Model (arch-CF5 closure)

#### Capability subtypes

Seven concrete `Capability` subtypes, all immutable records:

| Subtype | Constructor signature | Pattern-match target | Auto-allow conditions |
|---|---|---|---|
| `FileReadCap` | `FileReadCap(string Path)` | `fr.Path` | Path starts with `config.WorkingDirectory` |
| `FileWriteCap` | `FileWriteCap(string Path, string Operation="modify")` | `fw.Path` | None (always requires check unless session rule) |
| `ProcessExecCap` | `ProcessExecCap(string Binary, IReadOnlyList<string> Args)` | `pe.Binary` | Binary or "binary arg[0]" in safe-read-only list |
| `NetworkEgressCap` | `NetworkEgressCap(string Host, int Port=0, string Protocol="https")` | `ne.Host` | None |
| `VcsMutationCap` | `VcsMutationCap(string Repo, string Operation)` | `vc.Repo` | None (`push`/`force-push` not auto-blocked) |
| `MemoryCap` | `MemoryCap(string Namespace, string Operation)` | `mc.Namespace` | `Operation == "read"` |
| `AgentSpawnCap` | `AgentSpawnCap(string AgentType, string TaskSummary)` | `asc.AgentType` | None |

*observed fact ‚Äî `Permissions/Capability.cs`.*

#### Capability check evaluation order (`CheckCapabilitiesAsync`)

1. Empty capability list ‚Üí `Allowed = true` immediately; no prompt.
2. `_sessionDenyAll` contains `toolName` ‚Üí `Allowed = false` with session-block message (tells LLM not to suggest `chmod`/`chown`).
3. `_sessionAllowAll` contains `toolName` ‚Üí `Allowed = true` immediately.
4. For each capability: `_sessionDenyCapTypes` contains cap class name ‚Üí `Allowed = false`.
5. For each capability: session deny rules (`_sessionCapRules` with `allow=false`, matching cap type + pattern) ‚Üí `Allowed = false`.
6. For each capability: `_sessionAllowCapTypes` contains cap class name ‚Üí cap covered; skip to next cap.
7. For each capability: session allow rules (`_sessionCapRules` with `allow=true`, matching cap type + pattern) ‚Üí cap covered; skip to next cap.
8. For each capability: `IsAutoAllowedCapability` checks (¬ßCapability subtypes above) ‚Üí cap covered; skip to next cap.
9. Any uncovered capabilities ‚Üí `PromptUserForCapabilitiesAsync`.
10. User response:
    - `Allow` ‚Üí one-time allow; capability not memoized.
    - `Deny` ‚Üí `PermissionDeniedOnce` message.
    - `AllowAll` ‚Üí adds `toolName` to `_sessionAllowAll` AND all uncovered cap class names to `_sessionAllowCapTypes`.
    - `DenyAll` ‚Üí adds `toolName` to `_sessionDenyAll`.

*observed fact ‚Äî `PermissionEngine.cs:44-93`.*

#### Legacy permission check evaluation order (`CheckAsync`)

Used for tools that return an empty `RequiredCapabilities` list (most built-in tools not yet migrated to the capability model, and sub-agent tool dispatch).

1. Config deny rules for `toolName` (glob matched against full `input.ToString()`, case-insensitive) ‚Üí deny; may escalate to prompt after denial threshold.
2. `_sessionDenyAll` contains `toolName` ‚Üí deny (re-escalate to prompt after threshold).
3. `_sessionAllowAll` contains `toolName` ‚Üí allow.
4. `PermissionLevel.AutoAllow` (tool default) ‚Üí allow.
5. `PermissionLevel.Deny` (tool default) ‚Üí deny (may escalate to prompt after threshold).
6. Config allow rules for `toolName` (glob) ‚Üí allow.
7. Else ‚Üí `PromptUserAsync`.

*observed fact ‚Äî `PermissionEngine.cs:96-155`.*

#### Glob pattern semantics

- `*` alone ‚Üí matches any string.
- `*` in pattern ‚Üí `.*` in regex (match zero or more of any character).
- `?` in pattern ‚Üí `.` in regex (match exactly one character).
- All matching is case-insensitive.
- Patterns are anchored (`^...$`); no special directory separator handling.
- The `Ask` list in `ToolPermissionRules` is loaded and merged from config but is not evaluated in `CheckAsync` or `CheckCapabilitiesAsync` (present in schema; currently unused in evaluated logic).

*observed fact ‚Äî `PermissionEngine.MatchesPattern:350-358`, `AppConfig.ToolPermissionRules`.*

#### Hardcoded non-configurable blocks

| Category | Blocked values |
|---|---|
| Binaries (ProcessExecCap) | `sudo`, `su`, `doas`, `pkexec`, `chmod`, `chown`, `chattr`, `setfacl`, `icacls`, `takeown`, `attrib` |
| Write paths (FileWriteCap) | `/etc/`, `/usr/`, `/bin/`, `/sbin/`, `/System/`, `/Library/` |

*observed fact ‚Äî `PermissionEngine.IsBlockedBinary`, `PermissionEngine.IsProtectedPath`.*

#### Denial tracking

- 3 consecutive denials OR 20 total denials this session ‚Üí `_consecutiveDenials` triggers warn + escalate path (returns `true` from `TrackDenial`, which re-prompts instead of silently denying).
- Counter resets to 0 on any allow (`TrackAllow`).

*observed fact ‚Äî `PermissionEngine.TrackDenial:322-335`.*

**Owner:** `Permissions/PermissionEngine`, `Permissions/Capability`, `Permissions/PathGuard`, `Config/AppConfig.PermissionConfig`.

#### Turn budget interaction

- Sub-agent turn budget is internal to `AgentTool.ExecuteCoreAsync` and does **not** consume parent loop iterations.
- From the parent loop's perspective, the entire sub-agent execution is one tool call (one `ToolCall` in the batch).
- `AgentTool.IsConcurrencySafe = true` ‚Üí sub-agent can start as an in-flight task while the parent LLM stream is still active.
- Self-spawn is prevented: `Agent` tool is explicitly excluded from sub-agent tool registry (`if (tool.Name == "Agent") continue`). *observed fact ‚Äî `AgentTool.cs:43`.*

## High-Value Behaviors

### Cancellation and Abort

- **Ctrl+C once:** active `CancellationTokenSource` cancelled; in-flight LLM SSE stream aborted; tool tasks receive cancellation; `OperationCanceledException` in tool ‚Üí `ToolResult.Cancelled`. Session saved before exit. *observed fact ‚Äî `Program.cs:237-258`.*
- **Ctrl+C twice within 1.5 s:** `ProcessWatchdog.ScheduleHardKill()` + `Environment.Exit(0)` ‚Äî immediate hard-kill. *observed fact ‚Äî `Program.cs:253`.*
- **Sibling task abort:** if any parallel read-only tool returns `ResultClass.Crash`, `siblingAbortCts` is cancelled; remaining sibling tasks receive `ToolResult.Cancelled("sibling abort")`. *observed fact ‚Äî `ConversationLoop.cs:590-607`.*
- **Hook timeout:** each hook has a 30 s hard timeout (`CancelAfter(TimeSpan.FromSeconds(30))`); timeout ‚Üí warning logged, tool execution continues. *observed fact ‚Äî `HookRunner.cs:100`.*

### Streaming and Partial Output

- **TUI mode:** `ThinkingDelta` ‚Üí thinking panel; `TextDelta` ‚Üí streaming output panel in real time; `ToolCallDelta` ‚Üí tool dispatch queue; `Usage` ‚Üí `TokenTracker`.
- **During LLM stream:** `IsConcurrencySafe` read-only tools start executing as `inFlightTasks` before the stream ends. *observed fact ‚Äî `ConversationLoop.cs:233-239`.*
- **After stream ends:** remaining read-only tools complete via `Task.WhenAll`; writable tools execute serially after. *observed fact ‚Äî `ConversationLoop.cs:489-535`.*
- **Sub-agent streaming:** text from sub-agent's LLM is relayed to parent via `context.StreamText` if available (same output channel). *observed fact ‚Äî `AgentTool.cs:85-89`.*

### Doom-Loop Detection

| Field | Value |
|---|---|
| **Mechanism** | Three identical sequential tool-call sequences detected within a single turn |
| **Observable output** | Injected assistant message informing the LLM that a loop was detected; current turn ends |

- After every turn: `SessionManager.SaveAsync` appends tool results to JSONL. *observed fact ‚Äî `Program.cs` main event loop.*

## Security and Authorization

### Authentication

| Method | Where used | Storage |
|---|---|---|
| Bearer token (`LLAMA_API_KEY`) | Local llama-server requests (`Authorization: Bearer` header) | `docker/.env`, enforced `chmod 0600` |
| Provider API key | Cloud providers (Anthropic, OpenAI, Ollama) | `settings.json` under `providers.{name}.api_key`; no OAuth |
| Remote relay key (`OPENMONO_API_KEY`) | Tunnel / remote inference scenarios | Environment variable |

No OAuth, SSO, or session token mechanisms. All authentication is static API key injection. *observed fact ‚Äî `docs/CONFIG.md`, `AppConfig.ProviderSettings`, `docker-compose.yml:54`.*

### Authorization Model

Capability-based, not RBAC/ABAC. Capabilities are fine-grained (specific file path, specific binary, specific network host). Permission state is session-scoped in-memory; not persisted across sessions. See ¬ßPermission and Authorization Model for full evaluation order.

### Trust Boundaries

- LLM output (tool call arguments) is **untrusted**: all arguments pass through schema validation + `SanityCheck` before any side-effecting action.
- Workspace boundary enforced by `SanityCheck` (path-escape detection) and `PathGuard`; paths resolved relative to `config.WorkingDirectory` before checks.
- Interactive user input is **trusted** at the bash/shell level ‚Äî the system does not prevent shell injection in config hook `run` fields authored by the user.
- MCP tool inputs from external subprocesses are treated as tool call arguments and pass through the same validation pipeline.

### Secret Management

- API keys stored in `settings.json` (`chmod 0600`) or `docker/.env` (`chmod 0600`).
- No built-in key rotation or expiration.
- `SecretScanner` utility class is present (`Utils/`) but its integration point in the agent loop was not confirmed from source reading (see `contr-OQ1`).
- Keys are not logged in default mode; `--verbose` may surface them in SSE stream output. *strong inference.*
